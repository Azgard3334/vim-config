# Puml-parser
***
## Описание
***
Утилита для разбора PlantUML-диаграмм и последующего преобразования их в промежуточное представление. Разборщик обрабатывает `.puml / .plantuml / .uml файлы` и возвращает список сущностей и связей в JSON-формате.

__1. Разбор PlantUML-файлов__
***
Чтение `.puml / .plantuml / .uml файлы` файлов и извлечение из них структурных элементов диаграммы.

__2. Выделение сущностей и связей__
***
Определение:
* сущностей (классы, компоненты, акторы и т.д.)
* связей между ними (наследование, ассоциации, зависимости)

__3. Преобразование в JSON-представление__
***
Формирование единого промежуточного формата (JSON), удобного для дальнейшей обработки.

__4. Поддержка автоматической обработки__
***
Использование результата парсинга для:
* генерации кода
* анализа архитектуры
* преобразования в другие форматы

## Структура
***
__Папка `diagrams/`__
***
Содержит тестовые входные данные:
* файлы форматов `.puml .plantuml .uml`
* используются для проверки корректности работы парсера
* позволяют валидировать разбор различных конструкций (сущности, связи, пакеты и т.д.)

__Папка `grammar/`__
***
Содержит описание грамматики языка PlantUML:
* файл `plantuml.ebnf`
* реализован в формате EBNF

__Файл `puml_parser_logic.py`__
***
Основной модуль с логикой парсинга.
Отвечает за:
* загрузку и применение грамматики
* разбор входного .puml файла
* извлечение:
  * сущностей
  * связей
* преобразование результата в JSON-структуру

__Файл `puml_parser_cli.py`__
***
Командный интерфейс (CLI — Command Line Interface).
Отвечает за:
* запуск парсера из командной строки
* передачу входных параметров (например, путь к файлу)
* вывод результата (JSON) пользователю

Позволяет использовать парсер как отдельную утилиту.
